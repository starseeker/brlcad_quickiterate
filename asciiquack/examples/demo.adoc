== Document Structure

Headings and sections give your document a clear hierarchy.
asciiquack supports six heading levels with proportional sizes and
decorative rules under the top two levels.

=== Second-Level Section

Content under a second-level section appears here.
You can nest sections freely.

==== Third-Level Section

Deeper nesting is also supported, all the way down to level six.
At each level the font size decreases so the structure remains visually
clear.

== Text Formatting

inline markup lets you express emphasis and technical content:

* *Bold text* is rendered using the bold font variant.
* _Italic text_ is rendered using the italic font variant.
* *_Bold italic_* combines both variants.
* `Monospace text` is rendered using the monospace font variant.
* A link: https://asciidoc.org[AsciiDoc website]

== Tables

Tables present structured data in rows and columns.
When the first row is separated from the body by a blank line inside the
`|===` delimiters it is treated as a *header row* and rendered in bold with
a shaded background.
Column widths are proportional; a `cols` attribute of `"1,2,1"` makes the
middle column twice as wide as the flanking columns.

[cols="1,2,1"]
|===
| Tool | Description | Language

| asciiquack
| Converts AsciiDoc to PDF, HTML5, DocBook 5, and man pages
| C++17

| Asciidoctor
| The reference AsciiDoc processor and toolchain
| Ruby

| Pandoc
| Universal document converter supporting dozens of formats
| Haskell
|===

Inline markup such as *bold*, _italic_, and `monospace` works inside table cells
exactly as it does in body paragraphs.

== Code Blocks

Source code is set in the monospace font with a light grey background.

[source,python]
.A simple Python function
----
def greet(name: str) -> str:
    """Return a greeting string."""
    return f"Hello, {name}!"

if __name__ == "__main__":
    print(greet("world"))
----

Plain listing blocks work the same way:

----
$ asciiquack -b pdf examples/demo.adoc
asciiquack: converting 'examples/demo.adoc'
asciiquack: output written to 'examples/demo.pdf'
----

=== Font Embedding Details

Six TrueType font slots are embedded in the PDF output:

[horizontal]
`pdf-font`:: Regular body text (NotoSans-Regular)
`pdf-font-bold`:: Bold body text (NotoSans-Bold)
`pdf-font-italic`:: Italic body text (NotoSans-Italic)
`pdf-font-bold-italic`:: Bold-italic body text (NotoSans-BoldItalic)
`pdf-font-mono`:: Monospace text (NotoSansMono-Regular)
`pdf-font-mono-bold`:: Bold monospace text (NotoSansMono-Bold)

You can override any slot by passing the corresponding document attribute:

----
asciiquack -b pdf \
  -a pdf-font=/path/to/MyFont-Regular.ttf \
  -a pdf-font-bold=/path/to/MyFont-Bold.ttf \
  document.adoc
----

Only the slots you specify are overridden; the remaining slots continue
to use their defaults (Noto fonts when built from the repository, or the
PDF base-14 Helvetica and Courier families when the Noto fonts are not
available).
